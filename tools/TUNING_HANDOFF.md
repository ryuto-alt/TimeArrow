## 各ステージの成立要件(全ステージ共通)

## 既知の落とし穴(過去の検証で判明済み。必ず考慮)
- **RWの底打ち**: 対象時計は0で底打ち。閉門の再開判定は「時計-実効量 < closeT」。
  時計が古い門は1発(-10)で開かないが、`clock-10 < closeT` なら1発で開く。RWのみプランの反証は
  「必要RW本数 > 残数」(予算枯渇)で作るのが確実
- スラム門: FF経路(時計が若い)は1発、待ち経路(時計が古い)は2発になるよう closeT を選ぶと予算戦争が作れる
- 引き絞り中は世界0.25倍速(shot()が処理済み)。矢サイクルは shot() が処理済み
- 平地の振り子ノコは絶対通過不能 → 必ず pit_cross() を使う(待機位置が可動域+0.96の外であることも検証される)
- **矢の射程18**: 種まきFFは的まで18u以内から。ジオメトリ上の射点位置に注意
- 錠のFF: 開時刻が大きい錠は複数発必要(1発+10まで)。「種まき→他の作業→開く頃に到着」を意図した設計
- ボタン(S7 Button1)はFF矢のみで作動。RW矢は効かない=RWのみプランの構造的反証に使える
- ツタ(vine)はFFで即育つ。RWのみ/矢なしプランは growT まで待てるので、待ちが致命になる後続締切とセットで反証を作る

## 各ステージのビート(想定解の骨子)と反証の設計意図

### S1 (幅45, RW1) 2連橋
経路: x1→11(射点) FF+10→Target1(橋1,rise1=16) → 渡る[14-20橋-30] → x27射点 FF→Target2(橋2,rise2) → 渡る → 出口43。
反証: 矢なし=rise2(24)+走破 > limit。RWのみ=橋は上がるだけ。
knob: rise1/rise2/limit。橋2はrise2が大きく2発不可(RW1・矢1本なので単純に+10×1で届く時刻設計)。

### S2 (幅52, RW3) 四枚の閉門
GateA x14.6(スラム=必ず閉→RW) → GateB x22.5(スプリント=急げば開)→ GateC x35.5(スラム2) → GateD x43(スプリント2)。
反証: 矢なし/FFのみ=Aで死。想定解はRW2発(A,C)+スプリント2回、1発予備。
closeB/closeDは「A通過後に寄り道なく歩けば+1.5〜2.5s余裕」に。もたつくとRWで呼ぶ(予備1発の使い道)。

### S3 (幅58, RW3) 三つの錠の取引
射撃台x7.8 → Lock1 x10.6(open lock1: 待つ18 or FF払い) → Vine1 x13.5(育ち済=RW銀行) → スラムS1 x20.5 → スラムS2 x28.5 → Lock2 x38(歩きながらx22-28で種まき、射程18内) → GateZ x46.5(サンド: 待ち経路は閉じて2発必要) → 出口54。
反証: RWのみ=S1+S2+Zで4発>3。FFのみ=スラムで死。矢なし=スラムで死。
Z の closeZ は「FF種まき経路の到達 < closeZ」かつ「待ち経路の到達-10 ≥ closeZ」。

### S4 (幅64, RW3) サンド+動く壁
P1(x0-30): 現行と同じ: スラムA x13 → Lock4 x15.4(FF2発) → 刃ピットx19.4(pit_cross x0=16.6 x1=22.1) → GateB x24.4(サンド)。
P2: CW4(動く壁, x37から+1.0u/sでtravel15=x52まで) — FFゴーストで抜ける(通過必須) → 針山 → GateZ4 x59.5(サンド2: 壁が止まるまで待つ(startT+15s+α)とZが閉じ、-10でも開かない) → 出口62.5。
反証: RWのみ=B(上限10不足)。矢なし/FFのみ=スラムAで死。
sim では CW 通過を「FFを1発当ててghost通過(+2でよい, 実時間+0.4)」としてモデル化してよい。

### S5 (幅88, RW4) 3層
P1: リフトx12.5(arriveT=lift)をFF2発で降ろし→乗ってRW-10=上昇(refund 5, RW#1)。
P2デッキ[16,52]: スラムG x17.2(RW#2,若い時計1発) → 大玉Ball5(x19.5, rollT=ballRoll, speed=ballSpeed)が**東へ先行して道を塞ぐ**(追いつくと死)→ 退避ピット1 x28.6 に入り RW-10 で大玉を自分の背後(西)へ送る(RW#3) → LockD x46(歩きながらx30-40で種まき) → デッキ端52で降りる。
P3: Vine5 x61.2(growT=vineGrow)をFFで即育て→登る→最上層[62,88]: Ferry5 x72(振れ幅2.2,period=ferryP)で溝[70,74]を渡る → GateY5 x76.5(サンド: 待ち経路(ツタをvineGrowまで待つ)はYに間に合わずRW残数も枯渇) → LockZ5 x82(フェリー手前x66-70で種まき) → 出口85.5。
反証: 矢なし=リフト昇降不能。FFのみ=昇降不能(RW必須)。RWのみ=ツタが育つ前に…vineGrow+登攀+Y締切で構造的に不成立(closeY < vineGrow到達, かつRW予算枯渇)。
sim ではフェリー乗車=「wait(period/2以内の間合い)+乗る2.2u分」でモデル化してよい(振れ幅2.2、乗り待ちはperiod依存)。
